# 🏦 MDM IP Management System

Премиальная LegalTech-платформа для автоматизированного учета, мониторинга и финансовой оценки объектов интеллектуальной собственности (ИС). Проект объединяет в себе современные методы оценки, управление юридическими дедлайнами и базу знаний на базе ИИ.

---

## 🚀 Основные возможности

### 1. Финансовая оценка (AI Valuation)
- **Методология:** Использование метода **Relief from Royalty** (Освобождение от роялти) и **DCF** (Дисконтирование денежных потоков).
- **Интерактивность:** Глубокое интервью с ИИ-агентом для сбора специфических данных об активе.
- **Отчетность:** Мгновенная генерация PDF-отчета (Professional Valuation Report) с детализацией факторов влияния и аудитом данных.
- **Точность:** Учет отраслевых бенчмарков, юридической силы и рыночного охвата.

### 2. Управление портфелем ИС
- **Единый реестр:** Учет всех типов объектов (ПО, БД, Товарные знаки, Изобретения и др.).
- **Документооборот:** Хранение и привязка файлов к конкретным активам.
- **Генератор документов:** Автоматическое создание лицензионных договоров, NDA, претензий и исков на основе шаблонов.

### 3. Центр уведомлений и дедлайнов
- Автоматический расчет критических дат (продление патентов, оплата пошлин).
- Календарь предстоящих событий для исключения рисков потери прав.

### 4. Интеллектуальная база знаний
- Масштабируемая база статей по праву ИС.
- ИИ-ассистент для поиска ответов на юридические вопросы.

## 📊 Методология оценки (Valuation Methodology)

Алгоритм системы базируется на комбинированном подходе, интегрирующем классические финансовые модели с ИИ-анализом юридических рисков.

### 1. Нормативная база
Расчеты соответствуют международным и национальным стандартам:
- **International Valuation Standards (IVS 210, IVS 103)**.
- **Федеральный стандарт оценки №11** (Оценка НМА и интеллектуальной собственности).

### 2. Математическая модель (Income Approach)
Основной расчет производится по методу **Relief from Royalty (RfR)** с применением дисконтирования денежных потоков (DCF):

$$V = \sum_{t=1}^{n} \frac{Revenue_t \times Royalty\_Rate \times Legal\_Weight}{(1 + r)^t}$$

**Где:**
- $Revenue_t$ — прогнозная выручка от использования актива.
- $Royalty\_Rate$ — ставка условного роялти (определяется на основе отраслевых бенчмарков).
- $Legal\_Weight$ — коэффициент юридической устойчивости (рассчитывается ИИ на основе интервью).
- $r$ — ставка дисконтирования ($WACC + IP\_Risk\_Premium$).
- $n$ — горизонт прогнозирования (срок правовой охраны или экономической жизни).

### 3. Гибридный синтез весов
Система автоматически распределяет веса между **Доходным** (60–80%) и **Затратным** (20–40%) подходами в зависимости от:
- Наличия подтвержденных денежных потоков.
- Стадии жизненного цикла актива.
- Полноты исходных данных.

### 4. Раскрытие неопределенности (IVS 103)
В соответствии с принципом *Valuation Uncertainty*, система не выдает одну цифру, а формирует диапазон:
- **Min:** Консервативный сценарий (высокий риск).
- **Base Case:** Наиболее вероятное значение.
- **Max:** Оптимистичный сценарий (максимальная защита и охват).

### 5. Доказательный подход (Evidence-based)
Каждая оценка сопровождается "Аудиторским следом":
- Фиксация ответов пользователя в интервью.
- Протоколирование логики формирования коэффициентов.
- Сохранение версии алгоритма на момент расчета.

## 🛠 Технологический стек

- **Frontend:** Next.js 15 (React), Tailwind CSS, Lucide Icons, Recharts.
- **Backend:** FastAPI (Python 3.11), SQLAlchemy 2.0.
- **Database:** PostgreSQL.
- **AI Integration:** OpenRouter (LLM) для консультаций и расчетов.
- **Infrastructure:** Docker, Docker Compose, Nginx (для продакшн).

## 📦 Развертывание (Deployment)

### Способ 1: С помощью Docker (Рекомендуемый)

Это самый быстрый способ запустить систему со всеми зависимостями.

1.  **Настройка переменных окружения:**
    Создайте файл `.env` в корневом каталоге (можно скопировать из `.env.example`).
    ```ini
    DB_PASSWORD=your_secure_password
    SECRET_KEY=generate_random_string
    OPENROUTER_API_KEY=your_key_here
    ```

2.  **Запуск:**
    ```bash
    docker-compose up --build -d
    ```

3.  **Доступ:**
    - Приложение: `http://localhost:3000`
    - API (FastAPI): `http://localhost:8000/docs`

---

### Способ 2: Локальная установка (для разработки)

#### 1. Backend
```bash
cd backend
python -m venv .venv
source .venv/bin/activate  # Или .venv\Scripts\activate на Windows
pip install -r requirements.txt
uvicorn app.main:app --reload
```

#### 2. Frontend
```bash
cd frontend
npm install
npm run dev
```
