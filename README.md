# VKR-IP

`VKR-IP` — веб-система для учета, анализа и оценки объектов интеллектуальной собственности. Проект объединяет реестр объектов ИС, модуль стоимостной оценки, генерацию документов, базу знаний и контроль дедлайнов в одном интерфейсе.

## Что умеет система

- вести реестр объектов интеллектуальной собственности;
- хранить связанные документы и материалы;
- рассчитывать стоимость объектов ИС с использованием `Relief from Royalty` и `DCF`;
- формировать PDF-отчеты по результатам оценки;
- генерировать типовые юридические документы;
- отображать аналитику и обзорные показатели на dashboard;
- поддерживать базу знаний по тематике ИС;
- работать с дедлайнами и связанными сущностями проекта.

## Архитектура

Проект состоит из трех основных частей:

- `frontend` — клиентская часть на `Next.js 15` и `React 19`;
- `backend` — серверная часть на `FastAPI`, `SQLAlchemy`, `Pydantic`;
- `PostgreSQL` — основная база данных.

Для локального и production-запуска используются `Docker` и `Docker Compose`. На сервере production-окружение работает через `Nginx`.

## Локальный запуск через Docker

Для локального запуска используется файл `docker-compose.yml`.

### 1. Подготовка переменных окружения

Скопируйте шаблоны и заполните значения:

```bash
cp .env.example .env
cp backend/.env.example backend/.env
```

Минимально необходимо задать:

- `DB_PASSWORD`
- `DATABASE_URL`
- `SECRET_KEY`
- `OPENROUTER_API_KEY`

### 2. Запуск

```bash
docker-compose up --build -d
```

### 3. Доступ

- frontend: `http://localhost:3000`
- backend: `http://localhost:8000`
- backend docs: `http://localhost:8000/docs`

## Модуль оценки

Система оценки использует комбинированную логику:

- доходный подход через `Relief from Royalty`;
- дисконтирование денежных потоков (`DCF`);
- затратный подход как дополнительный ориентир;
- корректировки на юридические факторы, интервью и подтверждающие материалы;
- формирование диапазона значений `min / base / max`.

### Математическая модель оценки

Основной расчет стоимости объекта интеллектуальной собственности выполняется по методу освобождения от роялти (`Relief from Royalty, RfR`) с применением дисконтирования денежных потоков (`DCF`):

```math
V_{доход}=\sum_{t=1}^{n}\frac{Revenue_t \times RoyaltyRate \times K_{юр}}{(1+r)^t}
```

Где:

- `Vдоход` — стоимость объекта ИС по доходному подходу;
- `Revenue_t` — прогнозная годовая выручка от использования объекта ИС в период `t`;
- `RoyaltyRate` — ставка условного роялти, определяемая на основе отраслевых диапазонов и характеристик актива;
- `Kюр` — коэффициент юридической устойчивости объекта ИС;
- `r` — ставка дисконтирования;
- `t` — номер периода прогнозирования;
- `n` — горизонт прогнозирования в годах.

Ставка дисконтирования определяется как:

```math
r = BaseRate_{industry} + RiskPremium
```

Где:

- `BaseRateindustry` — базовая ставка для соответствующей отрасли;
- `RiskPremium` — премия за риск, формируемая с учетом результатов интервью, правовых факторов и анализа актива.

Коэффициент юридической устойчивости определяется как:

```math
K_{юр}=1+\sum_{i=1}^{m} LegalFactor_i
```

Где:

- `LegalFactori` — отдельные юридические факторы, влияющие на защищенность объекта;
- `m` — количество учтенных юридических факторов.

Дополнительно в системе рассчитывается стоимость по затратному подходу:

```math
V_{затрат}=R\&D \times 1.6
```

Где:

- `Vзатрат` — стоимость по затратному подходу;
- `R&D` — затраты на разработку, создание или доведение объекта до текущего состояния;
- `1.6` — коэффициент, учитывающий сопутствующие расходы, инфляцию и предпринимательскую надбавку.

Базовая итоговая стоимость определяется как комбинация доходного и затратного подходов:

```math
V_{base}=
\begin{cases}
0.9 \times V_{доход} + 0.1 \times V_{затрат}, & \text{если } Revenue > 1{\,}000{\,}000 \\
0.7 \times V_{доход} + 0.3 \times V_{затрат}, & \text{если } Revenue > 0 \\
V_{затрат}, & \text{если } Revenue = 0 \text{ и есть затраты}
\end{cases}
```

После этого базовая стоимость корректируется системой коэффициентов:

```math
V_{final}=V_{base}\times K_{юр}\times K_{int}\times K_{market}\times K_{jur}\times K_{brand}\times K_{dir}\times K_{lic}\times K_{enc}\times K_{prot}\times K_{sub}\times K_{purpose}
```

Где:

- `Vfinal` — итоговая стоимость объекта ИС;
- `Kint` — коэффициент, учитывающий подтвержденные ответы интервью;
- `Kmarket` — коэффициент рыночного охвата;
- `Kjur` — поправка на количество юрисдикций;
- `Kbrand` — поправка на силу бренда;
- `Kdir` — поправка на характер использования бренда;
- `Klic` — бонус за лицензионную активность;
- `Kenc` — дисконт за обременения;
- `Kprot` — бонус за подтвержденную правовую защищенность;
- `Ksub` — поправка на дополнительные метрики подтипа;
- `Kpurpose` — коэффициент цели оценки.

Диапазон итоговой оценки рассчитывается следующим образом:

```math
Spread = \max(0.05;\ 0.15-0.01 \times EvidenceCount)
```

```math
V_{min}=V_{final}\times(1-Spread)
```

```math
V_{max}=V_{final}\times(1+Spread)
```

Где:

- `Spread` — интервал неопределенности оценки;
- `EvidenceCount` — количество подтвержденных факторов и доказательств;
- `Vmin` — нижняя граница стоимости;
- `Vmax` — верхняя граница стоимости.

В текущей реализации оценка использует доказательный подход:

- ответы интервью сохраняются как подтвержденные факторы;
- связанные документы добавляются в `evidence_logs`;
- итоговый диапазон зависит от объема подтвержденной доказательной базы;
- отчет по оценке использует эти данные для интерпретации результата.
